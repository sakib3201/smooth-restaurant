## ADDED Requirements

### Requirement: Email templates are customizable and support variable replacement
The system SHALL provide default email templates for new reservation, confirmation, reminder, and cancellation events. Restaurant admins SHALL be able to customize the subject line and body of each template via the settings page. Templates SHALL support variable replacement for `{customer_name}`, `{party_size}`, `{reservation_date}`, `{reservation_time}`, `{restaurant_name}`, `{restaurant_phone}`, and `{status}`.

#### Scenario: Default email sent with variables replaced
- **WHEN** a new reservation is submitted
- **THEN** the customer receives an email with all template variables replaced with actual reservation data

#### Scenario: Admin customizes email template
- **WHEN** an admin updates the confirmation email body in settings and saves
- **THEN** subsequent confirmation emails use the customized template

#### Scenario: Plain text fallback is sent
- **WHEN** an email client does not render HTML
- **THEN** the system provides a plain text alternative generated by stripping HTML tags

---

### Requirement: Reminder emails are scheduled via WordPress Cron
The system SHALL register a twice-hourly cron hook that queries for confirmed reservations occurring within the next 2 hours where a reminder has not yet been sent. For each matching reservation, the system SHALL send the reminder email and mark the reservation as reminded.

#### Scenario: Reminder sent 2 hours before reservation
- **WHEN** the cron job runs at 17:00 and a confirmed reservation exists at 19:00 with no reminder sent
- **THEN** the system sends a reminder email to the customer and updates the reservation to mark reminder as sent

#### Scenario: No duplicate reminders sent
- **WHEN** the cron job runs again at 17:30 for the same 19:00 reservation
- **THEN** the system does not send a second reminder because the reservation is already marked as reminded

#### Scenario: Cancelled reservations do not receive reminders
- **WHEN** the cron job encounters a reservation with status `cancelled`
- **THEN** the system skips it and does not send a reminder
